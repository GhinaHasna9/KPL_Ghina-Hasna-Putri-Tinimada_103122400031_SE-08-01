Kode yang menjadi objek refaktorisasi adalah fungsi untuk menampilkan rincian pesanan di dalam sebuah modal. Fungsi tersebut bertugas mengambil data detail pesanan dari API berdasarkan `orderId`, kemudian menampilkan informasi tersebut ke dalam modal. ## Kode Awal

Berikut adalah kode awal sebelum dilakukan refaktorisasi:

```javascript
function fetchOrderDetails(orderId, token) {
    fetch(`https://example.com/api/order/${orderId}`, {
        headers: {
            'Authorization': token
        }
    })
    .then(response => {
        if (!response.ok) {
            throw new Error('Failed to fetch order details');
        }
        return response.json();
    })
    .then(order => {
        // Display order info
        const modal = document.getElementById('orderModal');
        const detailsDiv = modal.querySelector('#orderDetails');
        detailsDiv.innerHTML = '';

        const header = document.createElement('h3');
        header.textContent = `Order ID: ${order.id}`;
        detailsDiv.appendChild(header);

        const status = document.createElement('p');
        status.textContent = `Status: ${order.status}`;
        detailsDiv.appendChild(status);

        // Show modal
        modal.style.display = 'block';

        // Setup close button
        const closeBtn = modal.querySelector('.close');
        closeBtn.addEventListener('click', () => {
            modal.style.display = 'none';
        });

        // Setup confirm button
        const confirmBtn = modal.querySelector('#confirmOrderBtn');
        if (order.status === 'Delivered') {
            confirmBtn.style.display = 'none';
        } else {
            confirmBtn.addEventListener('click', () => {
                confirmOrder(order.id, token);
            });
        }
    })
    .catch(error => {
        console.error('Error:', error);
    });
}
```

## Identifikasi Masalah pada Kode Awal

Meskipun kode awal dapat berjalan dengan benar, terdapat sejumlah kekurangan yang perlu diperbaiki dari sisi clean code.

### 1. Satu Fungsi Menanggung Terlalu Banyak Tugas

Fungsi `fetchOrderDetails()` melakukan terlalu banyak hal sekaligus:

1. Mengirim permintaan ke API.
2. Memeriksa respons dari server.
3. Mengonversi data respons ke format JSON.
4. Mengakses elemen modal dari dokumen HTML.
5. Membangun elemen HTML untuk menampilkan detail pesanan.
6. Membuka modal ke layar.
7. Menyiapkan tombol close.
8. Menyiapkan tombol konfirmasi.
9. Menangani error.

Kondisi ini melanggar **Single Responsibility Principle**, yang menyatakan bahwa setiap fungsi sebaiknya hanya memiliki satu tugas utama.

---

### 2. Kode Sulit Dibaca dan Dipelihara

Karena semua proses dikerjakan dalam satu fungsi yang panjang, kode menjadi padat dan sulit ditelusuri. Mengubah tampilan modal mengharuskan penelusuran di tengah-tengah logika fetch API, dan sebaliknya.

---

### 3. Pengulangan Pola Pembuatan Elemen

Pembuatan elemen HTML dilakukan secara berulang dengan pola yang sama:

```javascript
const header = document.createElement('h3');
header.textContent = `Order ID: ${order.id}`;
detailsDiv.appendChild(header);

const status = document.createElement('p');
status.textContent = `Status: ${order.status}`;
detailsDiv.appendChild(status);
```

Setiap kali elemen baru perlu ditambahkan, pola ini harus ditulis ulang, melanggar prinsip **DRY (Don't Repeat Yourself)**.

---

### 4. Penggunaan Magic String

Nilai `"Delivered"` ditulis secara langsung dalam kondisi:

```javascript
if (order.status === 'Delivered') {
```

Nilai seperti ini disebut **magic string**. Jika string ini digunakan di banyak tempat dan suatu saat berubah, maka setiap kemunculannya harus ditemukan dan diubah satu per satu, yang rawan menyebabkan inkonsistensi.

---

### 5. Event Listener Berpotensi Menumpuk

Setiap kali `fetchOrderDetails()` dipanggil, kode menambahkan event listener baru pada tombol konfirmasi:

```javascript
confirmBtn.addEventListener('click', () => {
    confirmOrder(order.id, token);
});
```

Apabila modal dibuka beberapa kali, event listener pada tombol akan terus bertambah. Akibatnya, satu klik pada tombol bisa memicu pemanggilan `confirmOrder()` berkali-kali.

---

### 6. Tombol Konfirmasi Tidak Selalu Dipulihkan

Ketika status pesanan adalah `"Delivered"`, tombol konfirmasi disembunyikan:

```javascript
confirmBtn.style.display = 'none';
```

Namun kode tidak menjamin tombol tersebut akan ditampilkan kembali ketika modal dibuka untuk pesanan dengan status yang berbeda. Ini berpotensi menyebabkan tombol tetap tersembunyi secara tidak sengaja.

---

## Tujuan Refaktorisasi

Refaktorisasi dilakukan untuk mencapai beberapa hal berikut:

1. Memecah fungsi besar menjadi fungsi-fungsi kecil dengan tugas spesifik.
2. Memastikan setiap fungsi hanya memiliki satu tanggung jawab.
3. Menggunakan nama fungsi yang deskriptif dan ekspresif.
4. Menghilangkan pengulangan kode yang tidak perlu.
5. Mengganti magic string dengan konstanta.
6. Mencegah penumpukan event listener.
7. Menjamin konsistensi visibilitas elemen UI.

## Kode Setelah Refaktorisasi

```javascript
const API_BASE_URL = "https://example.com/api";

const ORDER_STATUS = {
  DELIVERED: "Delivered",
};

function getOrderModalElements() {
  const modal = document.getElementById("orderModal");

  return {
    modal,
    detailsContainer: modal.querySelector("#orderDetails"),
    closeButton: modal.querySelector(".close"),
    confirmButton: modal.querySelector("#confirmOrderBtn"),
  };
}

async function fetchOrderDetails(orderId, token) {
  try {
    const order = await getOrderById(orderId, token);
    showOrderDetailsModal(order, token);
  } catch (error) {
    handleOrderDetailsError(error);
  }
}

async function getOrderById(orderId, token) {
  const response = await fetch(`${API_BASE_URL}/order/${orderId}`, {
    headers: {
      Authorization: token,
    },
  });

  if (!response.ok) {
    throw new Error("Failed to fetch order details");
  }

  return response.json();
}

function showOrderDetailsModal(order, token) {
  const {
    modal,
    detailsContainer,
    closeButton,
    confirmButton,
  } = getOrderModalElements();

  renderOrderDetails(detailsContainer, order);
  setupCloseButton(closeButton, modal);
  setupConfirmButton(confirmButton, order, token);
  openModal(modal);
}

function renderOrderDetails(container, order) {
  container.innerHTML = "";

  const header = createTextElement("h3", `Order ID: ${order.id}`);
  const status = createTextElement("p", `Status: ${order.status}`);

  container.appendChild(header);
  container.appendChild(status);
}

function createTextElement(tagName, text) {
  const element = document.createElement(tagName);
  element.textContent = text;
  return element;
}

function setupCloseButton(closeButton, modal) {
  closeButton.onclick = () => closeModal(modal);
}

function setupConfirmButton(confirmButton, order, token) {
  confirmButton.onclick = null;

  if (order.status === ORDER_STATUS.DELIVERED) {
    hideElement(confirmButton);
    return;
  }

  showElement(confirmButton);

  confirmButton.onclick = () => {
    confirmOrder(order.id, token);
  };
}

function openModal(modal) {
  modal.style.display = "block";
}

function closeModal(modal) {
  modal.style.display = "none";
}

function showElement(element) {
  element.style.display = "block";
}

function hideElement(element) {
  element.style.display = "none";
}

function handleOrderDetailsError(error) {
  console.error("Error:", error.message);
}
```

## Penjelasan Perubahan Setelah Refaktorisasi

### 1. Konstanta untuk URL dan Status Pesanan

```javascript
const API_BASE_URL = "https://example.com/api";

const ORDER_STATUS = {
  DELIVERED: "Delivered",
};
```

URL API dan status pesanan dipisahkan ke dalam konstanta di luar fungsi. Apabila suatu saat URL atau nama status berubah, modifikasi cukup dilakukan di satu tempat tanpa perlu menelusuri seluruh kode.

---

### 2. Fungsi `getOrderModalElements()`

```javascript
function getOrderModalElements() {
  const modal = document.getElementById("orderModal");

  return {
    modal,
    detailsContainer: modal.querySelector("#orderDetails"),
    closeButton: modal.querySelector(".close"),
    confirmButton: modal.querySelector("#confirmOrderBtn"),
  };
}
```

Fungsi ini memusatkan pengambilan elemen HTML modal. Fungsi lain tidak perlu mengulang query selector yang sama, sehingga jika ID atau class elemen berubah, cukup ubah di satu tempat.

---

### 3. Fungsi `fetchOrderDetails()` yang Lebih Ringkas

```javascript
async function fetchOrderDetails(orderId, token) {
  try {
    const order = await getOrderById(orderId, token);
    showOrderDetailsModal(order, token);
  } catch (error) {
    handleOrderDetailsError(error);
  }
}
```

Fungsi utama kini hanya bertanggung jawab mengkoordinasikan alur: ambil data, tampilkan modal, tangani error. Penggunaan `async/await` membuat alur lebih mudah dibaca dibandingkan chaining `.then()` yang panjang.

---

### 4. Fungsi `getOrderById()` untuk Isolasi Logika API

```javascript
async function getOrderById(orderId, token) {
  const response = await fetch(`${API_BASE_URL}/order/${orderId}`, {
    headers: {
      Authorization: token,
    },
  });

  if (!response.ok) {
    throw new Error("Failed to fetch order details");
  }

  return response.json();
}
```

Seluruh logika komunikasi dengan API dikapsulasi dalam fungsi ini. Perubahan pada endpoint atau header tidak akan menyentuh kode tampilan.

---

### 5. Fungsi `showOrderDetailsModal()` sebagai Orkestrator UI

```javascript
function showOrderDetailsModal(order, token) {
  const { modal, detailsContainer, closeButton, confirmButton } = getOrderModalElements();

  renderOrderDetails(detailsContainer, order);
  setupCloseButton(closeButton, modal);
  setupConfirmButton(confirmButton, order, token);
  openModal(modal);
}
```

Fungsi ini mengatur tampilan modal secara terstruktur. Setiap langkah didelegasikan ke fungsi spesifik sehingga mudah dilacak.

---

### 6. Fungsi `renderOrderDetails()` dan `createTextElement()`

```javascript
function renderOrderDetails(container, order) {
  container.innerHTML = "";

  const header = createTextElement("h3", `Order ID: ${order.id}`);
  const status = createTextElement("p", `Status: ${order.status}`);

  container.appendChild(header);
  container.appendChild(status);
}

function createTextElement(tagName, text) {
  const element = document.createElement(tagName);
  element.textContent = text;
  return element;
}
```

Pola pembuatan elemen yang berulang diekstrak ke `createTextElement()`, menghilangkan duplikasi kode. Container juga dikosongkan terlebih dahulu untuk memastikan data lama tidak tersisa.

---

### 7. Fungsi `setupConfirmButton()` yang Aman dari Penumpukan

```javascript
function setupConfirmButton(confirmButton, order, token) {
  confirmButton.onclick = null;

  if (order.status === ORDER_STATUS.DELIVERED) {
    hideElement(confirmButton);
    return;
  }

  showElement(confirmButton);

  confirmButton.onclick = () => {
    confirmOrder(order.id, token);
  };
}
```

Penugasan `onclick = null` di awal memastikan handler lama dihapus sebelum handler baru ditambahkan, mencegah penumpukan event. Selain itu, `showElement()` dipanggil secara eksplisit agar tombol selalu dipulihkan ketika pesanan tidak berstatus `"Delivered"`.

---

### 8. Fungsi Utilitas untuk Visibilitas dan Modal

```javascript
function openModal(modal) { modal.style.display = "block"; }
function closeModal(modal) { modal.style.display = "none"; }
function showElement(element) { element.style.display = "block"; }
function hideElement(element) { element.style.display = "none"; }
```

Operasi manipulasi `display` yang berulang dienkapsulasi dalam fungsi kecil bernama deskriptif, sehingga kode pemanggil lebih mudah dibaca.

## Prinsip Clean Code yang Diterapkan

### 1. Single Responsibility Principle

Setiap fungsi memiliki satu tugas yang jelas dan terfokus:

| Fungsi                      | Tanggung Jawab                          |
| --------------------------- | --------------------------------------- |
| `getOrderById()`            | Mengambil data pesanan dari API         |
| `renderOrderDetails()`      | Menampilkan detail pesanan ke container |
| `setupCloseButton()`        | Mengonfigurasi tombol close modal       |
| `setupConfirmButton()`      | Mengonfigurasi tombol konfirmasi        |
| `handleOrderDetailsError()` | Menangani dan mencatat error            |

---

### 2. Meaningful Name

Nama setiap fungsi dipilih agar mencerminkan perannya secara langsung:

```javascript
getOrderById()
showOrderDetailsModal()
renderOrderDetails()
setupConfirmButton()
createTextElement()
```

Pembaca dapat memahami alur program hanya dari nama fungsi tanpa harus membaca implementasinya secara mendetail.

---

### 3. Avoid Magic String

Status `"Delivered"` tidak lagi ditulis langsung di dalam kondisi, melainkan disimpan dalam konstanta:

```javascript
const ORDER_STATUS = {
  DELIVERED: "Delivered",
};
```

---

### 4. DRY (Don't Repeat Yourself)

Pola pembuatan elemen HTML yang berulang diekstrak ke satu fungsi yang dapat digunakan kembali:

```javascript
function createTextElement(tagName, text) {
  const element = document.createElement(tagName);
  element.textContent = text;
  return element;
}
```

---

### 5. Prevent Duplicate Event Handler

Penugasan `onclick = null` sebelum handler baru memastikan tidak ada event yang menumpuk:

```javascript
confirmButton.onclick = null;
```

---

### 6. Consistent UI State

Tombol konfirmasi selalu dikembalikan ke kondisi terlihat sebelum diputuskan apakah perlu disembunyikan, memastikan state UI selalu konsisten di setiap pembukaan modal.

## Perbandingan Kode Awal dan Kode Refaktorisasi

| Aspek                 | Kode Awal                     | Kode Refaktorisasi                        |
| --------------------- | ----------------------------- | ----------------------------------------- |
| Ukuran fungsi utama   | Satu fungsi panjang           | Dipecah menjadi banyak fungsi kecil       |
| Tanggung jawab        | Semua tercampur               | Masing-masing terpisah dan terfokus       |
| Keterbacaan           | Sulit karena panjang          | Mudah karena setiap fungsi ringkas        |
| Magic string          | Ada (`'Delivered'`)           | Diganti konstanta `ORDER_STATUS.DELIVERED`|
| Event listener        | Berpotensi menumpuk           | Aman dengan `onclick = null`              |
| Visibilitas tombol    | Tidak selalu konsisten        | Selalu dipulihkan secara eksplisit        |
| Kemudahan pemeliharaan| Sulit jika ada perubahan      | Lebih mudah karena setiap bagian terisolasi|
| Reusability           | Rendah                        | Tinggi (`createTextElement`, dll.)        |

---

## Kesimpulan

Refaktorisasi pada kode ini dilakukan untuk mengatasi masalah utama yaitu satu fungsi yang menanggung terlalu banyak tanggung jawab sekaligus. Dengan memecah kode menjadi fungsi-fungsi kecil yang masing-masing memiliki tujuan tunggal, kode menjadi jauh lebih mudah dibaca, diuji, dan dikembangkan.

Penerapan konstanta untuk menggantikan magic string, penggunaan `async/await` untuk menggantikan promise chain, serta pencegahan penumpukan event listener adalah beberapa perbaikan konkret yang dihasilkan dari proses refaktorisasi ini.

Secara keseluruhan, kode hasil refaktorisasi memiliki struktur yang lebih bersih dan lebih mencerminkan prinsip clean code, meskipun perilaku fungsionalnya tetap sama dengan kode awal — menampilkan detail pesanan dalam modal dan memungkinkan pengguna untuk mengonfirmasi pesanan.
