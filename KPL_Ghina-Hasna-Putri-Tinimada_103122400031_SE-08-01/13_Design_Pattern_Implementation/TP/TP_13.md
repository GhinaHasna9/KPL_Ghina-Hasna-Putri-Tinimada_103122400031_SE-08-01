## Deskripsi Tugas

Tugas ini meminta mahasiswa untuk membuka repository project tugas besar, kemudian mengidentifikasi satu design pattern yang diterapkan di dalamnya. Design pattern yang dipilih bebas, selama implementasinya dapat ditunjukkan melalui kode program dan dapat dijelaskan desain serta cara kerjanya.

Repository yang dianalisis adalah project **Foodora Catering App**, yaitu aplikasi berbasis web untuk pemesanan layanan katering. Aplikasi ini dibangun menggunakan **React.js** di sisi frontend, **Node.js + Express.js** di sisi backend, dan **MySQL** sebagai sistem manajemen basis data.

Setelah menelusuri kode pada repository tersebut, ditemukan penerapan **Singleton Pattern** pada bagian yang menangani koneksi ke database.

---

## Design Pattern yang Ditemukan

Design pattern yang berhasil diidentifikasi dalam project ini adalah:

```txt
Singleton Pattern
```

Singleton Pattern adalah pola desain yang membatasi pembuatan suatu objek hanya satu kali, kemudian objek tersebut digunakan secara bersama oleh bagian-bagian lain dalam aplikasi yang membutuhkannya.

Pada project Foodora Catering App, Singleton Pattern diterapkan pada file konfigurasi koneksi database, yaitu:

```txt
backend/db.js
```

File tersebut bertugas membuat satu objek `pool` untuk mengelola koneksi ke database MySQL. Objek `pool` kemudian diekspor dan dimanfaatkan oleh file lain, terutama `backend/server.js`, untuk menjalankan berbagai query.

---

## Alasan Memilih Singleton Pattern

Singleton Pattern dipilih karena koneksi ke database tidak semestinya dibuat dari awal setiap kali ada endpoint yang membutuhkannya atau setiap kali ada request dari pengguna.

Apabila setiap endpoint membuat koneksi database secara mandiri, aplikasi akan memboroskan resource sistem dan berisiko mengalami gangguan akibat terlalu banyak koneksi yang dibuka secara bersamaan. Dengan menerapkan Singleton Pattern, aplikasi cukup membuat satu connection pool di awal, lalu pool tersebut digunakan kembali oleh seluruh proses backend sepanjang aplikasi berjalan.

Dalam ekosistem Node.js, Singleton Pattern secara alami dapat diterapkan melalui sistem modul. Ketika sebuah file diekspor menggunakan `module.exports` dan kemudian diimpor menggunakan `require()`, Node.js akan menyimpan hasilnya dalam cache modul. Artinya, objek yang sama akan dikembalikan setiap kali `require()` dipanggil untuk file yang sama.

## Kode Implementasi Singleton Pattern

### File `backend/db.js`

```javascript
const mysql = require("mysql2/promise");
require("dotenv").config();

const pool = mysql.createPool({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  port: process.env.DB_PORT || 3306,
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0,
});

module.exports = pool;
```

## Kode Penggunaan Singleton

### File `backend/server.js`

```javascript
const pool = require("./db");
```

Baris di atas mengimpor objek `pool` yang telah dibuat di `db.js`. Karena Node.js menggunakan cache modul, objek yang sama persis yang dikembalikan setiap kali `require("./db")` dipanggil dari manapun.

Contoh penggunaan `pool` pada endpoint pengecekan koneksi:

```javascript
app.get("/api/health", async (req, res) => {
  try {
    const [rows] = await pool.query("SELECT 1 AS status");

    res.json({
      success: true,
      message: "Database terhubung",
      data: rows[0],
    });
  } catch (error) {
    res.status(500).json({
      success: false,
      message: "Database gagal terhubung",
      error: error.message,
    });
  }
});
```

Contoh penggunaan `pool` pada endpoint pembuatan pesanan:

```javascript
app.post("/api/pesanan", async (req, res) => {
  const connection = await pool.getConnection();

  try {
    const { id_user, items, metode_pembayaran, tanggal_booking } = req.body;

    if (!id_user || !Array.isArray(items) || items.length === 0) {
      return res.status(400).json({
        success: false,
        message: "Silakan login dan pilih minimal 1 menu sebelum membuat pesanan.",
      });
    }

    await connection.beginTransaction();

    // proses query database dilakukan di sini

    await connection.commit();
  } catch (error) {
    await connection.rollback();

    res.status(500).json({
      success: false,
      message: error.message,
    });
  } finally {
    connection.release();
  }
});
```

## Penjelasan Desain

Pada file `backend/db.js`, objek connection pool dibuat menggunakan fungsi `mysql.createPool()`:

```javascript
const pool = mysql.createPool({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  port: process.env.DB_PORT || 3306,
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0,
});
```

Konfigurasi di dalam `createPool()` mencakup parameter koneksi seperti alamat host, nama pengguna, kata sandi, nama database, nomor port, serta batas jumlah koneksi yang dapat dibuka secara bersamaan.

Setelah objek `pool` terbentuk, objek tersebut diekspor keluar modul:

```javascript
module.exports = pool;
```

Dengan cara ini, file lain di dalam proyek tidak perlu membuat koneksi database baru secara mandiri. Cukup dengan menulis:

```javascript
const pool = require("./db");
```

maka objek `pool` yang sama langsung dapat digunakan untuk menjalankan query maupun mengambil koneksi dari pool.

Pada endpoint `/api/health`, pool digunakan untuk menjalankan query verifikasi:

```javascript
const [rows] = await pool.query("SELECT 1 AS status");
```

Sementara pada endpoint `/api/pesanan`, pool digunakan untuk mengambil koneksi yang mendukung transaksi:

```javascript
const connection = await pool.getConnection();
```

Dengan demikian, satu objek pool yang didefinisikan di `db.js` dapat melayani kebutuhan seluruh endpoint di `server.js` tanpa membuat koneksi baru setiap saat.

## Alur Kerja Singleton Pattern pada Project

Diagram alur penerapan Singleton Pattern pada project ini:

```txt
backend/db.js
    ↓
Membuat satu objek pool koneksi database
    ↓
module.exports = pool
    ↓
backend/server.js mengambil pool dengan require("./db")
    ↓
Seluruh endpoint backend memanfaatkan objek pool yang sama
```

Dengan alur ini, koneksi database dikelola secara terpusat dan tidak perlu dibuat ulang untuk setiap request.

## Analisis Komponen Design Pattern

Singleton Pattern pada project ini memiliki beberapa elemen kunci:

1. **Single Instance**
   Objek `pool` hanya dibuat satu kali, yaitu saat file `backend/db.js` pertama kali dimuat.

2. **Global Access Point**
   Objek `pool` dapat diakses oleh file mana pun melalui `require("./db")` tanpa perlu membuat instance baru.

3. **Reusable Connection**
   Pool yang sama digunakan berulang kali oleh semua endpoint di `backend/server.js`.

4. **Efisiensi Sumber Daya**
   Aplikasi tidak membuka koneksi database baru secara manual di setiap endpoint, sehingga resource sistem dapat digunakan lebih optimal.

5. **Sentralisasi Konfigurasi**
   Seluruh pengaturan koneksi database hanya berada di satu file, memudahkan pemeliharaan dan perubahan konfigurasi.

---

## Keunggulan Penerapan Singleton Pattern

Penerapan Singleton Pattern pada lapisan koneksi database membawa sejumlah manfaat nyata:

1. Koneksi database lebih efisien karena memanfaatkan satu connection pool yang dikelola bersama.
2. Konfigurasi database tersentralisasi di satu file sehingga lebih mudah dikelola.
3. Setiap endpoint tidak perlu mendefinisikan ulang parameter koneksi.
4. Kode backend menjadi lebih bersih dan terstruktur.
5. Perubahan pada konfigurasi database cukup dilakukan di `db.js` tanpa menyentuh file lain.

---

## Catatan Implementasi

Perlu dicatat bahwa implementasi Singleton Pattern pada project ini tidak menggunakan struktur class Singleton seperti yang umum dijumpai dalam paradigma Object Oriented Programming. Namun secara konsep, prinsipnya tetap sama: objek `pool` hanya dibuat satu kali di dalam modul `db.js`, dan setiap pemanggilan `require("./db")` akan mengembalikan objek yang persis sama berkat mekanisme caching modul Node.js.

Oleh karena itu, pendekatan ini dapat disebut sebagai **module-based Singleton** atau Singleton berbasis sistem modul yang menjadi idiom umum dalam ekosistem Node.js.

---

## Kesimpulan

Berdasarkan analisis terhadap repository Foodora Catering App, ditemukan penerapan **Singleton Pattern** pada file `backend/db.js`.

Design pattern ini digunakan untuk memastikan bahwa hanya ada satu objek connection pool database yang dibuat selama aplikasi berjalan. Objek tersebut dibuat menggunakan `mysql.createPool()`, diekspor melalui `module.exports`, dan diimpor kembali di `server.js` menggunakan `require("./db")`.

Mekanisme cache modul Node.js memastikan objek yang sama dikembalikan setiap kali `require()` dipanggil, sehingga prinsip Singleton terpenuhi tanpa perlu implementasi class khusus. Hasilnya, koneksi database menjadi lebih terkelola, lebih efisien, dan lebih mudah dipelihara sepanjang siklus hidup aplikasi.
