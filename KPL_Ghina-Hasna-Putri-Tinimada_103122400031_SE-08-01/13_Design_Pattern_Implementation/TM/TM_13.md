## Event Delegation sebagai Pola Penanganan Event dalam JavaScript

**Kelas:** SE-08-01

Event delegation merupakan sebuah teknik pengelolaan event pada JavaScript di mana event listener ditempatkan pada elemen induk, bukan pada tiap elemen anak secara individual. Dengan pendekatan ini, sebuah event listener tunggal mampu menangani berbagai elemen sekaligus tanpa perlu mendaftarkan listener secara terpisah untuk masing-masing elemen.

Dalam JavaScript, event seperti `click`, `keyup`, atau `submit` pada umumnya didaftarkan langsung ke elemen yang bersangkutan. Bayangkan ada 10 tombol dalam halaman — pendekatan konvensional akan memasang satu event listener pada setiap tombol tersebut. Cara ini tidak efisien, terutama ketika jumlah elemen sangat banyak atau ketika elemen-elemen tersebut baru muncul setelah halaman selesai dimuat.

Event delegation bekerja dengan memanfaatkan mekanisme **event bubbling**. Event bubbling adalah perilaku alami browser di mana sebuah event yang terjadi pada elemen anak akan merambat naik ke elemen-elemen induknya. Misalnya, ketika pengguna mengklik sebuah tombol di dalam sebuah `div`, event klik tersebut pertama kali dipicu pada tombol, kemudian merambat ke `div` induknya, lalu terus naik ke elemen-elemen di atasnya.

Berkat mekanisme bubbling tersebut, kita dapat mendaftarkan event listener cukup pada elemen induk saja, lalu menentukan elemen mana yang sebenarnya memicu event menggunakan properti `event.target`.

Berikut contoh sederhana HTML yang akan digunakan:

```html
<ul id="daftar-menu">
  <li>Nasi Goreng</li>
  <li>Mie Ayam</li>
  <li>Bakso</li>
</ul>
```

Pendekatan konvensional memasang listener pada setiap elemen `li`:

```javascript
const items = document.querySelectorAll("li");

items.forEach(function(item) {
  item.addEventListener("click", function() {
    console.log("Menu dipilih:", item.textContent);
  });
});
```

Meski kode di atas berfungsi, setiap elemen `li` memiliki listener sendiri. Jika terdapat puluhan item, memori yang digunakan pun semakin besar. Lebih jauh lagi, item baru yang ditambahkan secara dinamis tidak akan otomatis mendapatkan listener.

Dengan event delegation, cukup satu listener pada elemen `ul`:

```javascript
const daftarMenu = document.getElementById("daftar-menu");

daftarMenu.addEventListener("click", function(event) {
  if (event.target.tagName === "LI") {
    console.log("Menu dipilih:", event.target.textContent);
  }
});
```

Pada kode tersebut, listener dipasang di elemen `ul`. Saat pengguna mengklik salah satu `li`, event bubbling membawa event tersebut ke `ul`. Program kemudian memeriksa apakah elemen yang diklik adalah `LI` menggunakan `event.target.tagName`, lalu mengeksekusi logika yang sesuai.

Komponen kunci dalam teknik ini adalah properti `event.target`, yang menunjuk ke elemen asli yang memicu event. Dengan demikian, meskipun listener berada di elemen induk, kita tetap dapat mengetahui elemen anak mana yang berinteraksi dengan pengguna.

Event delegation banyak diterapkan dalam aplikasi web modern, misalnya pada daftar produk yang panjang, kolom komentar, tabel data, menu navigasi dinamis, atau deretan tombol aksi yang dibuat dari data server.

Contoh penerapan lain menggunakan atribut `data-*`:

```html
<div id="panel-aksi">
  <button data-action="edit">Edit</button>
  <button data-action="delete">Hapus</button>
  <button data-action="detail">Detail</button>
</div>
```

```javascript
const panelAksi = document.getElementById("panel-aksi");

panelAksi.addEventListener("click", function(event) {
  const aksi = event.target.dataset.action;

  if (aksi === "edit") {
    console.log("Tombol Edit diklik");
  } else if (aksi === "delete") {
    console.log("Tombol Hapus diklik");
  } else if (aksi === "detail") {
    console.log("Tombol Detail diklik");
  }
});
```

Pada contoh di atas, satu listener pada `panel-aksi` mampu membedakan ketiga tombol berdasarkan nilai atribut `data-action` masing-masing.

Event delegation dapat dipandang sebagai salah satu bentuk design pattern dalam JavaScript karena menyediakan solusi yang terstruktur dan berulang untuk permasalahan umum dalam pengelolaan event. Alih-alih mendaftarkan banyak listener untuk tiap elemen, kita menggunakan satu listener terpusat pada elemen induk dan membiarkan event bubbling bekerja. Pola ini menghasilkan kode yang lebih ramping, hemat memori, dan mudah dipelihara.

Keunggulan event delegation antara lain:

1. Lebih hemat karena hanya membutuhkan satu event listener untuk banyak elemen.
2. Secara alami mendukung elemen yang dibuat secara dinamis.
3. Kode lebih singkat dan lebih mudah dikelola.
4. Penggunaan memori lebih efisien.
5. Menyederhanakan penanganan event pada struktur DOM yang besar.

Meski demikian, penerapan event delegation perlu dilakukan dengan cermat. Penting untuk memastikan bahwa hanya event dari elemen yang dimaksud saja yang ditangani. Untuk itu, biasanya digunakan pengecekan melalui `event.target`, `tagName`, atribut `data-*`, atau metode `matches()`.

Contoh pengecekan menggunakan `matches()`:

```javascript
document.addEventListener("click", function(event) {
  if (event.target.matches(".btn-hapus")) {
    console.log("Tombol hapus diklik");
  }
});
```

Kode di atas hanya akan mengeksekusi logika jika elemen yang diklik memiliki class `btn-hapus`, sehingga listener yang dipasang di level document tidak akan terpicu oleh sembarang klik.

Kesimpulannya, event delegation adalah teknik dalam JavaScript untuk mengelola event secara efisien dengan memanfaatkan event bubbling. Daripada mendaftarkan listener pada setiap elemen anak, kita cukup memasang satu listener di elemen induk dan menentukan elemen yang diklik menggunakan `event.target`. Teknik ini merupakan pola penting dalam pengembangan web modern karena menghasilkan kode yang lebih efisien, fleksibel, dan mudah dikembangkan, khususnya ketika berhadapan dengan banyak elemen atau elemen yang bersifat dinamis.
